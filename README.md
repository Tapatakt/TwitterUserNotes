# Twitter User Notes

Плагин для Chrome/Brave, который позволяет оставлять личные комментарии (заметки) к пользователям Twitter / X.

> ⚠️ **Важно:** плагин навайбкожен.

## Что умеет

- При наведении на имя пользователя через 0.5 секунд справа экрана появляется текстовое поле с твоим комментарием.
- **Заголовок тултипа** — ник пользователя (`@UserName`).
- Если комментарий раньше не сохранялся — поле пустое.
- **Enter** или потеря фокуса — сохраняет комментарий.
- **Ctrl** (пока тултип открыт) — закрепляет его. Закреплённый тултип не исчезает при уходе мыши и закрывается только:
  - по клику снаружи,
  - по Enter,
  - при наведении на другого пользователя.
- Пустой комментарий при сохранении удаляет запись из хранилища.

## Как определяется пользователь

Плагин использует гибридный подход:

1. **Сначала** ищет элемент `<span>` или `<a>`, чей текст строго равен `@UserName`.  
   Это работает для упоминаний в тексте твитов и ретвитов, где стандартная ссылка ведёт на твит, а не на профиль.
2. **Если текст не подошёл** — проверяет `href` ближайшей ссылки (`x.com/UserName` или `twitter.com/UserName`).  
   Это ловит имена в шапках твитов и конструкции вроде «UserName нравится ваш ответ».

Исключены системные пути: `home`, `explore`, `notifications`, `tos`, `privacy`.

## Установка (ручная)

1. Открой `chrome://extensions/` (или `brave://extensions/`).
2. Включи **Режим разработчика** (переключатель справа сверху).
3. Нажми **Загрузить распакованное расширение** (Load unpacked).
4. Выбери папку с этим проектом (где лежит `manifest.json`).

## Структура

```
├── manifest.json                 — манифест расширения
├── README.md                     — этот файл
├── .gitignore                    — исключения для Git
└── content-scripts/
    ├── storage.js                — работа с chrome.storage.local
    ├── tooltip.js                — UI тултипа (справа экрана, с заголовком)
    ├── link-detector.js          — обнаружение пользователей (текст + ссылка)
    └── content.js                — точка входа, связка модулей
```

## Технические детали

- **Manifest V3**
- Content script работает только на `x.com` и `twitter.com`
- Хранение — `chrome.storage.local` (без SQL, без синхронизации)
- События мыши ловятся в фазе захвата (`capture: true`), чтобы встроенные скрипты Twitter не перехватывали их раньше нас
- Поддерживаются как обычные твиты, так и ретвиты
